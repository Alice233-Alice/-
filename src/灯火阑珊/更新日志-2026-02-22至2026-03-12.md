# 踏月寻仙更新日志

统计区间：2026-02-22 至 2026-03-12

说明：
- 本日志基于当前工作区文件结构、模块内容与文件修改时间整理。
- 当前目录不是 Git 仓库，因此以下内容是按现有实现反推的阶段性更新摘要，不是逐 commit 精确回放。
- 仅聚焦 `前端状态栏` 与 `MVU 变量结构`，不展开纯世界书文案、开局文本、生图提示词等内容。

## 2026-02-22

### 前端状态栏
- 新增 `region-utils.ts`，建立地域归一化与层级推断基础能力。
- 支持把 `地层/天层/下层` 这类粗粒度层级，与 `神州/东苍/南炎/北冥/天渊/归墟` 等具体域名区分开来。
- 为后续头部区域、地图页、行踪一致性修复提供统一的地域判断逻辑。

### 变量结构
- 为后续 `本尊.行踪.当前区域` 与 `本尊.行踪.所属层级` 的双字段模型打下兼容基础。

## 2026-03-02

### 前端状态栏
- 引入主题系统与主题状态存储，支持消息楼层级别的主题持久化。
- 状态栏开始通过 CSS 变量驱动配色、边框、按钮、页脚与背景纹样，而不是写死样式。
- 增补多套预设主题，状态栏视觉从单一配色扩展到可切换皮肤。

### 变量结构
- 主题选择结果写入消息变量 `ui_theme_id`，前端界面开始持久化自身 UI 偏好。

## 2026-03-04 至 2026-03-05

### 前端状态栏
- 状态栏开始围绕 `本尊/任务/声望/战斗/渡劫` 的完整 MVU 数据模型渲染，而不再只依赖零散字段。
- 修炼体系显示开始使用统一境界与阈值计算逻辑，为后续预设面板、红颜境界显示提供同一套口径。

### 变量结构
- 拆分并完善 `schema/protagonist.ts`、`schema/systems.ts`、`schema/common.ts`。
- `本尊` 结构补齐：`行踪`、`身份`、`背包`、`法宝`、`杂物袋`、`战斗状态`、`当前敌人`、`渡劫状态`。
- `任务列表` 引入容错映射，统一任务类型到 `主线/支线/每日/临危受命/秘境探索`，统一任务状态到 `进行中/已完成/已失败`。
- `声望系统` 引入自动关系推导，可根据声望值自动得到 `中立/友好/敌对/不死不休` 等关系描述。
- 新增 `可参与机遇`、`当前处境`、`_系统设置.启用行动提示`，为后续行动提示面板提供结构支撑。
- 新增 `难度系统` 与 `_难度系统内部数据`，状态栏后续可直接显示天道感应、压力档位与内部结算快照。

## 2026-03-07

### 前端状态栏
- 数据读取层升级为更强健的 Pinia Store。
- 增加“最后一次有效快照”机制，当前楼层变量缺失时可回退到聊天级稳定数据，而不是直接回初始值。
- 增加“向前继承最近有效楼层”的兜底逻辑，减少初始楼层或隐藏楼层造成的空白状态栏。
- 增加消息轮询与事件双通道刷新，降低 MVU 事件漏触发时的显示失步概率。
- 增加根据叙事文本修复临时战斗/渡劫状态的能力，例如战斗结束后强制脱战、劫云散去后关闭渡劫态。
- 增加行踪一致性修复，AI 漏写所属层级或把具体地点退化成粗粒度地域时，前端会自动纠偏。

### 变量结构
- `schema.ts` 作为总入口统一组合 `world / protagonist / characters / systems` 各子模块。
- `红颜` 数据开始接入更强的归一化与快照机制，为后续一体双魂与好感度限幅做准备。

## 2026-03-08

### 前端状态栏
- 主界面新增 `图鉴` 与 `行动` 两个主要页签，状态栏从纯数值面板扩展为“状态 + 行动 + 卡片”的复合 UI。
- 新增 `ActionMenuPanel.vue`：
  - 展示 `当前处境` 与 `可参与机遇`。
  - 支持开启/关闭行动提示生成。
  - 支持手动刷新机遇列表。
  - 支持将行动描述自动整理成玩家输入文本并注入酒馆输入框。
  - 支持“随缘而行”随机行动兜底。
  - 对红颜类行动增加 `双修/亲密/调情` 子类识别与专属视觉。
- 页脚改版为三段式信息区：
  - 左侧显示行动入口与可选行动数徽标。
  - 中间显示世界时钟，并按时辰推断昼夜图标。
  - 右侧显示难度感应、难度档位、趋势箭头与环境警告。
- 状态栏默认首屏切到 `图鉴`，前端重心从单纯数据展示转向更强的叙事承载。

### 变量结构
- `可参与机遇` 正式进入主流程，包含 `名称/来源/类型/描述/回报预期/风险评估/时限/优先级`。
- `难度系统` 的显示字段开始被前端消费，形成“变量直接驱动 UI 提示”的闭环。

## 2026-03-11

### 前端状态栏
- 新增 `PresetPanel.vue` 开局预设面板：
  - 可视化编辑本尊的等级、修为、灵根、体质、功法、本命兵器、灵石、身份、位置。
  - 可视化管理神通、法宝、背包、红颜初始关系。
  - 支持预设导入与导出。
  - 支持直接写回当前楼层 MVU 数据。
- `CompanionsPanel.vue` 升级为按等级自动换算境界描述，并按境界颜色渲染红颜卡片。
- 增加 `MvuGuardBootstrap.ts` 并在入口启动：
  - 自动修正 `./任务列表`、`/红颜/...` 等路径格式到标准路径。
  - 自动把 `虞汐/虞颜` 的别名路径归并到 `虞汐颜`。
  - 自动把 `小成/中成/大圆满` 等熟练度别名归一为 `熟练/精通/圆满`。
  - 在变量更新结束后重新用总 `Schema` 过一遍，前端侧补上一层 MVU 纠错守卫。
- 红颜、图鉴、预设、行动、难度等多个面板已经形成统一 Store 出口。

### 变量结构
- `schema/constants.ts` 将大境界扩展到 48 级链路，覆盖 `练气` 到 `仙帝`。
- `schema/characters.ts` 扩充并固化红颜角色库默认数据，前端预设与图鉴可直接读取静态角色库。
- `schema.ts` 加入一体双魂归一化：
  - 将 `虞汐`、`虞颜` 合并到 `虞汐颜`。
  - 合并双魂快照，避免好感或等级在前端显示为两份分裂数据。
- `schema.ts` 加入红颜好感度快照 `_好感度快照`，将单次变动幅度限制在合理范围内，减少 AI 一回合暴涨暴跌。

## 2026-03-12

### 前端状态栏
- `MapPanel.vue` 补上任务标题显示兜底：
  - 优先显示任务 `名称`。
  - 若名称为空，则从 `任务ID` 中提取中文语义。
  - 若仍提取不到中文，则按任务类型显示 `主线任务/支线任务/每日任务` 等中文默认名。
- 进一步避免前端直接暴露 `MQ/SQ/task_01` 之类英文任务代号。

### 变量结构
- `schema.ts` 对任务名做统一清洗：
  - 自动剥离 `MQ/SQ/task_01/main_quest` 一类英文前缀。
  - 任务名为空时优先回退到任务 ID 中的中文部分。
  - 若任务 ID 也无中文语义，则回退为中文类型名。
- `[mvu_update]变量更新规则.yaml` 与 `[mvu_update]变量输出格式-极简版.yaml` 收紧任务输出规范：
  - 明确禁止英文、拼音、缩写、编号型任务 ID 与名称。
  - 明确给出错误示例 `/任务列表/MQ_疗伤结缘` 与正确示例 `/任务列表/疗伤结缘`。
  - 目标是从提示词层就阻止英文任务名进入 MVU 数据。
