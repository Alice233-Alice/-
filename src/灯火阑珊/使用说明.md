## 实时更新机制

### 问题根源

之前状态栏不能实时更新的原因是：
1. 没有完整的 MVU 变量结构定义（只有 TypeScript 类型，没有 zod schema）
2. 没有注册变量结构的脚本
3. store.ts 没有使用 `Schema.parse()` 来解析和验证数据
4. 缺少正确的 MVU 变量监听

### 解决方案

现在的实现：

1. **完整的 zod schema**：[`schema.ts`](schema.ts:60) 定义了完整的变量结构，包括：
   - 自动计算衍生字段（突破阈值、寿元上限、境界描述等）
   - 数据验证和转换（数值范围限制、空物品自动清理等）
   - 默认值设置（使用 `.prefault()`）

2. **变量结构注册**：[`脚本/变量结构/index.ts`](脚本/变量结构/index.ts) 将 schema 注册到 MVU 框架

3. **响应式数据解析**：[`store.ts`](store.ts:98) 使用 `Schema.parse()` 解析数据：
   ```typescript
   rawData.value = Schema.parse(statData);
   ```

4. **MVU 事件监听**：[`store.ts`](store.ts:152) 监听 `Mvu.events.VARIABLE_UPDATE_ENDED` 事件实时更新

## 使用步骤

### 1. 部署脚本

将构建生成的脚本上传并在酒馆中加载：
```
dist/美化状态栏/脚本/变量结构/index.js
```

### 2. 使用 initvar 初始化变量

在世界书中创建 `[initvar]变量初始化勿开` 条目，复制 [`世界书/变量/initvar.yaml`](世界书/变量/initvar.yaml) 的内容。

**多开局兼容性**：
- 模板提供了最小化的初始值
- 所有字段都有默认值，不会因缺少字段而报错
- 你只需修改关键字段（如主角姓名、体质、灵根等）
- 其他字段会自动使用默认值

示例：设置主角为"太虚龙剑体"

```yaml
本尊:
  体质: 太虚龙剑体  # 只需修改这一行
  # 其他字段保持默认值即可
```

### 3. 通过 `/setvar` 测试实时更新

在酒馆聊天中输入：
```
/setvar key=本尊.体质 value=太虚龙剑体
```

状态栏应该立即更新显示新的体质！

### 4. AI 自动更新变量

AI 可以在回复中使用 MVU 命令更新变量：
```
{{mvu 本尊.修为+50}}
{{mvu 本尊.体质=先天道体}}
```

## 常见问题

### Q: 状态栏显示的还是默认值？

A: 检查：
1. 脚本是否加载（查看 Console 是否有"MVU 变量结构已注册"）
2. initvar 是否正确初始化（使用 `/echo {{getvar::本尊.体质}}` 查看）
3. MVU 框架是否加载（等待几秒后刷新页面）

### Q: 如何添加新角色？

A: 在 initvar.yaml 的 `红颜` 或 `NPC图鉴` 部分添加：

```yaml
红颜:
  角色名:
    等级: 1
    修为: 0
    灵根: 冰灵根
    体质: 玄冰体
    # ... 其他必填字段
    好感度: 0
    关系: 陌生人
    重要性: 路人
```

### Q: 能否自定义更多字段？

A: 可以！修改 [`schema.ts`](schema.ts) 添加新字段，然后：
1. 运行 `pnpm build` 生成新的 schema.json
2. 在 initvar.yaml 中添加对应字段
3. 重新部署脚本

## 技术细节

### 为什么使用 `.prefault()` 而不是 `.default()`？

`.prefault()` 是 zod 4.x 的新特性，在解析时会：
1. 如果字段不存在，使用默认值
2. 如果字段存在但为 `undefined`，也使用默认值
3. 更好地支持增量更新

### 为什么物品数量为 0 时会自动删除？

```typescript
.transform(data => _.pickBy(data, ({ 数量 }) => 数量 > 0))
```

这样可以：
1. 保持数据清洁
2. 减少存储空间
3. 避免显示空物品

### 衍生字段是如何计算的？

[`schema.ts:38`](schema.ts:38) 的 `addDerivedFields` 函数会在每次解析数据时自动计算：
- 突破阈值（根据等级查表）
- 寿元上限（根据等级查表）
- 境界描述（等级转换为文字）
- 状态（根据修为和突破状态判断）
- 进度百分比（修为/突破阈值 * 100%）

这确保了显示数据始终与基础数据保持同步！
