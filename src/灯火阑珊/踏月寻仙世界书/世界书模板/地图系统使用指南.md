# 踏月寻仙 - 地图系统使用指南

## 🎯 系统概览

地图系统由三个部分协同工作：

1. **Schema 数据层** ([`schema.ts`](../schema.ts)) - 存储地图的数值数据
2. **世界书描述层** ([`地图-优化版.yaml`](地图-优化版.yaml)) - 提供文字描述和背景
3. **动态管理层** ([`动态世界书管理脚本`](../脚本/动态世界书管理/index.ts)) - 智能激活相关条目

---

## 📦 快速开始

### 第一步：初始化地图数据

从 [`initvar-地图完整版.yaml`](../世界书/变量/initvar-地图完整版.yaml) 复制需要的区域到你的 `initvar.yaml`：

```yaml
世界地图:
  神州:
    layer: "地层"
    danger: 10
    desc: "修仙界的中心大陆"
    connections: ["北冥", "南炎", "西庚", "东苍"]
  
  # 根据需要添加更多区域...
```

### 第二步：配置世界书

将 [`地图-优化版.yaml`](地图-优化版.yaml) 的内容复制到你的世界书中，设置：

- **条目一**（世界空间结构）：🔵 蓝灯 @D [系统]在深度 深度1 顺序15
- **条目二到五**：🟢 绿灯 @D [AI]在深度 深度3 顺序60-63

### 第三步：启用动态管理脚本

确保 [`动态世界书管理`](../脚本/动态世界书管理/index.ts) 脚本已加载并设置为自动加载。

脚本会根据 `本尊.行踪.当前区域` 自动激活相关地图条目。

---

## 🗺️ 数据结构说明

### 世界地图字段

```typescript
世界地图: {
  [区域名: string]: {
    layer: '天层' | '地层' | '下层',     // 所属层级
    danger: number,                      // 危险度 0-100
    desc: string,                        // 简短描述
    connections: string[]                // 连接的区域列表
  }
}
```

### 行踪字段

```typescript
本尊.行踪: {
  当前区域: string,        // 当前所在区域名
  所属层级?: string,       // 所属层级（可选）
  环境描述?: string,       // 环境描述（可选）
  危险度?: number,         // 危险度（可选）
  可用通道?: string[],     // 可用通道（可选）
  导航信息?: string        // 导航信息（可选）
}
```

## 🎮 使用场景

### 场景1：角色移动到新区域

**AI 输出**：
```
{{mvu 本尊.行踪.当前区域="南炎"}}
{{mvu 本尊.行踪.所属层级="地层"}}
{{mvu 本尊.行踪.危险度=60}}
```

**效果**：
- 动态世界书脚本自动将 `[地图] 地层大陆` 改为蓝灯
- `[地图] 南炎` 相关条目也会被激活
- AI 获得南炎的详细信息，生成更贴切的描述

### 场景2：发现新区域

**AI 输出**：
```
{{mvu 世界地图.秘境山谷={layer:"地层",danger:50,desc:"隐藏的灵药谷",connections:["东苍"]}}}
{{mvu 本尊.行踪.当前区域="秘境山谷"}}
```

**效果**：
- 新区域被添加到地图数据中
- 角色移动到新区域
- 可在后续对话中继续使用

### 场景3：跨层级冒险

**从地层进入下层**：
```
{{mvu 本尊.行踪.当前区域="黄泉古迹"}}
{{mvu 本尊.行踪.所属层级="下层"}}
{{mvu 本尊.行踪.危险度=90}}
```

**效果**：
- 动态脚本激活 `[地图] 下层禁地` 条目
- AI 获得下层的危险信息和描述
- 自动调整叙事风格以匹配环境

## 🔧 动态管理规则

动态世界书脚本使用以下逻辑：

### 规则1：层级触发

```typescript
// 当在天层时，激活天层地图条目
当前层级 === '天层' → 激活 [地图] 天层区域
```

### 规则2：区域触发

```typescript
// 当在某区域或相邻区域时，激活该区域条目
当前区域 === '神州' → 激活 [地图] 地层大陆
当前区域 === '北冥' || 最近提到'北冥' → 激活 [地图] 北冥
```

### 规则3：关键词触发

```typescript
// 当最近消息提到相关关键词时激活
提到('海'|'航'|'船') → 激活 [地图] 四海区域
提到('归墟'|'黄泉'|'幽冥') → 激活 [地图] 下层禁地
```

### 规则4：智能预加载

```typescript
// 当在某区域时，预加载相邻区域
当前区域 === '神州' → 激活神州 + 预加载(北冥, 南炎, 西庚, 东苍)
```

## 💡 最佳实践

### 1. 初始化策略

**最小化初始化**（推荐新手）：
```yaml
世界地图:
  神州:
    layer: "地层"
    danger: 10
    desc: "修仙界的中心大陆"
    connections: []
```

**完整初始化**（推荐完整体验）：
- 复制 [`initvar-地图完整版.yaml`](../世界书/变量/initvar-地图完整版.yaml) 的全部内容

### 2. 命名规范

- ✅ 区域名应简短明确：`神州`、`北冥`、`南炎`
- ✅ 世界书条目命名：`[地图] 区域名`
- ❌ 避免过长或含特殊字符的名称

### 3. 连接关系

**定义连接时考虑双向性**：
```yaml
神州:
  connections: ["北冥", "南炎", "西庚", "东苍"]

北冥:
  connections: ["神州"]  # 反向连接
```

### 4. 危险度设定

```
0-20:   安全区（宗门、城镇）
20-40:  低危区（外围森林、矿区）
40-60:  中危区（深入探索区）
60-80:  高危区（禁地边缘）
80-95:  极危区（禁地核心）
95-100: 绝地（归墟、炎渊）
```

## 🐛 故障排除

### Q: 地图条目没有自动激活？

**检查清单**：
1. ✅ 动态世界书管理脚本是否正确加载？（查看 Console）
2. ✅ `本尊.行踪.当前区域` 是否设置正确？
3. ✅ 世界书条目命名是否符合规则？（`[地图] xxx`）
4. ✅ 条目是否设置为绿灯 @D [AI]在深度？

### Q: Token 没有明显节省？

**可能原因**：
1. 太多条目被设为蓝灯
2. 规则太宽松，导致过多条目被激活
3. 开启 `DEBUG: true` 查看哪些条目被激活

### Q: AI 不知道某个区域的信息？

**解决方案**：
1. 检查该区域是否在 `世界地图` 中初始化
2. 检查对应的世界书条目是否被激活（查看 Console）
3. 尝试在消息中明确提到该区域，触发关键词激活

## 🚀 高级用法

### 1. 动态添加区域

AI 可以在对话中发现并添加新区域：

```
{{mvu 世界地图.神秘洞府={
  layer:"地层",
  danger:55,
  desc:"你发现的隐秘洞府，灵气浓郁",
  connections:["东苍"]
}}}
```

### 2. 修改区域数据

随着剧情发展修改危险度：

```
{{mvu 世界地图.神州.danger=30}}  // 魔族入侵，危险度上升
```

### 3. 记录探索历史

使用 `世界图志` 记录重大事件：

```
{{mvu 世界图志.天渊异动={
  状态:"警戒中",
  事件:"近期天渊坠物频繁，西庚损失惨重"
}}}
```

### 4. 与状态栏联动

状态栏会自动显示：
- 当前区域名称
- 所属层级
- 危险度（带颜色）
- 可用通道

## 🎯 总结

地图系统的三层架构：

1. **数据层（MVU 变量）** → 存储客观数据
2. **描述层（世界书）** → 提供丰富描述
3. **管理层（动态脚本）** → 智能节省 Token

通过这种设计，你可以：
- ✅ 用少量 Token 维护庞大的世界地图
- ✅ 让 AI 根据位置自动获取环境信息
- ✅ 随时扩展新区域而不影响性能
- ✅ 在状态栏实时显示位置信息
