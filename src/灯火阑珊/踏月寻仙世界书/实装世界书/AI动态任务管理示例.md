# AI 动态任务管理系统

## 📋 核心理念

**初始状态：** `任务列表: {}` （空对象）  
**运行方式：** AI根据剧情发展自动添加、更新、完成任务

---

## 🎯 工作流程

### 1. 初始化（空任务列表）

在 `initvar.yaml` 中设置：

```yaml
任务列表: {}  # 开局时没有任务
声望系统: {}  # 开局时没有声望记录
```

### 2. AI动态添加任务

AI会在以下场景自动添加任务：

#### 场景A：NPC发布任务
```
师父："你已入门剑阁，先去执事堂领取份例吧。"

<UpdateVariable>
任务列表.领取份例:
  类型: 每日
  目标: 前往外门执事堂领取本月修炼资源
  状态: 进行中
</UpdateVariable>
```

#### 场景B：触发剧情任务
```
你在藏书阁发现了一本古老的剑谱，上面记载着失传的剑意...

<UpdateVariable>
任务列表.寻找剑意传承:
  类型: 支线
  目标: 探索古剑谱的秘密，寻找剑意传承
  状态: 进行中
</UpdateVariable>
```

#### 场景C：主线剧情推进
```
白清弦："想要在修仙界立足，必须突破筑基境界。"

<UpdateVariable>
任务列表.突破筑基:
  类型: 主线
  目标: 积累修为到100，准备突破筑基境界
  状态: 进行中
</UpdateVariable>
```

#### 场景D：临时紧急任务
```
警报响起！妖兽攻山！所有弟子立即集合！

<UpdateVariable>
任务列表.抵御兽潮:
  类型: 临危受命
  目标: 协助宗门抵御妖兽进攻
  状态: 进行中
</UpdateVariable>
```

### 3. AI更新任务状态

#### 任务进行中 → 已完成
```
你成功领取了本月的修炼资源：灵石x100，培元丹x3

<UpdateVariable>
任务列表.领取份例.状态: 已完成
本尊.灵石: +100
本尊.背包.培元丹:
  名称: 培元丹
  描述: 增加修为的丹药
  品阶: 玄阶下品
  数量: 3
</UpdateVariable>
```

#### 任务进行中 → 已失败
```
你在洗剑池修炼时走火入魔，任务失败了...

<UpdateVariable>
任务列表.洗剑池修炼.状态: 已失败
本尊.修为: -20
</UpdateVariable>
```

### 4. AI删除或清理任务

完成的任务可以保留在列表中作为记录，也可以删除：

```yaml
# 方式1：保留记录
任务列表.领取份例.状态: 已完成

# 方式2：删除任务（MVU支持删除操作）
<UpdateVariable>
<remove>任务列表.领取份例</remove>
</UpdateVariable>
```

## 📝 AI任务管理规则总结

基于 `[mvu_update]变量更新规则.yaml` 第31行：

```yaml
任务列表: 接取添加状态=进行中；完成/失败更新；避免大于3主线
```

### AI应遵循的规则：

1. **添加任务**
   - 初始状态总是 `状态: 进行中`
   - 根据剧情自然添加，不要一次性添加太多
   - 主线任务不超过3个

2. **更新任务**
   - 只更新 `状态` 字段（进行中 → 已完成/已失败）
   - 不要修改 `类型` 和 `目标`

3. **任务类型使用**
   - `主线`：核心剧情，最多3个
   - `支线`：可选任务，数量不限
   - `每日`：可重复任务
   - `临危受命`：突发紧急任务

4. **任务完成奖励**
   - 完成任务时同时更新奖励（灵石、物品、声望等）

## ✅ 推荐的初始化配置

### 开局空任务列表（推荐）

```yaml
任务列表: {}
声望系统: {}
```

**优点：**
- 完全由AI根据剧情动态生成
- 更自然，更符合开放世界体验
- AI有更大的创作自由度

### 开局预设少量引导任务（可选）

```yaml
任务列表:
  熟悉宗门:
    类型: 支线
    目标: 探索剑阁，熟悉宗门环境
    状态: 进行中

声望系统:
  剑阁:
    值: 0
    关系: 新入门弟子
```

**优点：**
- 给玩家明确的初始目标
- 适合新手玩家

## 🎨 进阶玩法

### 1. 任务链系统

AI可以设计连续任务：

```
任务A完成 → 触发任务B → 触发任务C
```

### 2. 隐藏任务

某些特殊条件触发的隐藏任务：

```
条件：好感度达到50 + 在特定地点
触发：隐藏剧情任务
```

### 3. 时限任务

通过 `目标` 字段标注时限：

```yaml
任务列表.紧急救援:
  类型: 临危受命
  目标: "三天内赶到百花谷救援（剩余2天）"
  状态: 进行中
```
