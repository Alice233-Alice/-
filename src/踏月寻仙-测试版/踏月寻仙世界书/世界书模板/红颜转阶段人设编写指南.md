# 红颜转阶段人设编写指南

## 概念说明

**转阶段人设**是指红颜角色随着好感度提升、剧情发展，其对{{user}}的态度、对话风格、行为模式会发生阶段性变化。这让角色更立体，情感线更自然。

## 核心设计原则

### 1. 好感度阶段划分

推荐使用 5 阶段划分法：

| 阶段 | 好感度范围 | 关系状态 | AI 行为特征 |
|------|-----------|---------|------------|
| **陌生期** | -20 ~ 20 | 陌生人/初识 | 保持距离，客套礼貌或冷淡 |
| **好感期** | 21 ~ 40 | 朋友/好友 | 开始信任，愿意交流 |
| **心动期** | 41 ~ 60 | 好友→暗生情愫 | 频繁关注，偶有心跳 |
| **确认期** | 61 ~ 80 | 情侣/深情 | 明确心意，亲密互动 |
| **深爱期** | 81 ~ 100 | 至爱/生死相依 | 全心全意，不离不弃 |

### 2. 转阶段触发机制

**MVU 变量**：
```yaml
红颜.凌寒镜:
  好感度: 45  # 当前好感度
  关系: "朋友"  # 当前关系状态（对应好感度阶段）
  重要性: "重要角色"
```

**世界书条目命名**：
```
[角色] 凌寒镜 - 陌生期 (好感度 -20~20)
[角色] 凌寒镜 - 好感期 (好感度 21~40)
[角色] 凌寒镜 - 心动期 (好感度 41~60)
[角色] 凌寒镜 - 确认期 (好感度 61~80)
[角色] 凌寒镜 - 深爱期 (好感度 81~100)
```

**触发条件设置**：
- 使用世界书的"变量条件"功能
- 设置为：`红颜.凌寒镜.好感度 >= 41 AND 红颜.凌寒镜.好感度 <= 60`

## 实战案例：凌寒镜（清冷师姐型）

### 阶段 1：陌生期（-20 ~ 20）

**关系状态**：同门师姐 → 陌生同门

**性格表现**：
- 清冷疏离，不苟言笑
- 心镜映照下只是公事公办
- 偶尔指点，但语气冰冷

**对话风格**：
```
❌ 避免：亲昵称呼、关心细节、主动接近
✅ 使用：
  - 称呼：师弟、你
  - 语气：简短、客观、疏离
  - 例句："修炼一途，需脚踏实地。"
```

**世界书条目示例**：
```yaml
---

【当前关系】同门师姐，保持距离

【态度】
  - 清冷疏离，公事公办
  - 心镜映照{{user}}，发现心性纯正，但仅此而已
  - 偶尔指点修炼，语气冰冷客观
  - 不会主动接近，不关心{{user}}私事

【对话风格】
  - 称呼：师弟、你
  - 语气：简短、客观、疏离
  - 禁止：亲昵称呼、关心细节、主动靠近
  - 例句："修炼一途，需脚踏实地。" "你的剑意尚显稚嫩。"

【行为模式】
  - {{user}}有难：视情况出手，事后不多言
  - {{user}}请教：冷淡指点，点到为止
  - 日常互动：礼貌但保持距离

【好感度变化条件】
  - 真诚待人 +3
  - 修炼勤奋 +2  
  - 虚伪欺骗 -10
  - 背叛宗门 -50

Keys: 凌寒镜, 陌生期, 好感度-20到20
Condition: 红颜.凌寒镜.好感度 >= -20 AND 红颜.凌寒镜.好感度 <= 20
```

### 阶段 2：好感期（21 ~ 40）

**性格表现**：

  - 温和指点："这个诀窍，你且记住..."

```

Keys: 凌寒镜, 好感期, 好感度21到40
Condition: 红颜.凌寒镜.好感度 >= 21 AND 红颜.凌寒镜.好感度 <= 40
```

## 实现方式

### 方式一：多条目分阶段（推荐）

**优点**：
- 清晰分离，易于管理
- AI 只看到当前阶段，不会混淆
- 便于调整单个阶段

**缺点**：
- 条目数量多
- 需要配置变量条件

**实现**：
```
[角色] 凌寒镜 - 陌生期 (Condition: 好感度 <= 20)
[角色] 凌寒镜 - 好感期 (Condition: 好感度 21~40)
[角色] 凌寒镜 - 心动期 (Condition: 好感度 41~60)
[角色] 凌寒镜 - 确认期 (Condition: 好感度 61~80)
[角色] 凌寒镜 - 深爱期 (Condition: 好感度 >= 81)
```

### 方式二：单条目全阶段

**优点**：
- 条目少，易于总览
- 全部设定在一处

**缺点**：
- 内容长，消耗 token
- AI 可能混淆阶段

**实现**：
```yaml
---
[角色] 凌寒镜 - 全阶段

根据当前好感度调整态度：
  
【-20~20 陌生期】
  - 清冷疏离...
  
【21~40 好感期】
  - 开始关注...
  
【41~60 心动期】
  - 频繁偶遇...
  
【61~80 确认期】
  - 确认心意...
  
【81~100 深爱期】
  - 生死相依...

⚠️ 请根据当前好感度值只使用对应阶段的设定！
```

### 方式三：动态世界书管理（最优）

使用动态世界书脚本，根据好感度自动启用/禁用对应阶段条目：

```typescript
// 在 脚本/动态世界书管理/index.ts 中添加
{
    entryPattern: /^\[角色\]\s*凌寒镜\s*-\s*陌生期/,
    shouldEnable: ctx => {
        const favor = ctx.mvuData?.红颜?.['凌寒镜']?.好感度 ?? 0;
        return favor >= -20 && favor <= 20;
    },
    priority: 8,
},
{
    entryPattern: /^\[角色\]\s*凌寒镜\s*-\s*好感期/,
    shouldEnable: ctx => {
        const favor = ctx.mvuData?.红颜?.['凌寒镜']?.好感度 ?? 0;
        return favor >= 21 && favor <= 40;
    },
    priority: 8,
},
// ... 其他阶段
```

## 最佳实践

### 1. 阶段过渡要自然

❌ 错误：好感度 60→61 突然从傲娇变温柔
✅ 正确：在 58-62 之间设置过渡期，逐渐转变

### 2. 保持角色核心性格

所有阶段都要保持角色的核心特质：
- 凌寒镜：清冷 → 始终清冷，只是程度不同
- 焰绯月：傲娇 → 始终傲娇，只是逐渐坦诚

### 3. 添加"转阶段触发事件"

在关键好感度节点设计特殊事件：
- 好感度 40→41：心动触发事件
- 好感度 60→61：表白事件
- 好感度 80→81：生死相依事件

### 4. 好感度变化要合理

```yaml
【好感度变化示例】
  - 真诚待人：+2~5（日常积累）
  - 危难相救：+10~20（重大事件）
  - 虚伪欺骗：-10~30（严重影响）
  - 背叛伤害：-50~100（关系破裂）
```

### 5. 利用"关系"字段同步更新

```yaml
# 在 MVU 变量更新规则中添加
红颜.凌寒镜.关系:
  check:
    - 根据好感度自动更新：
      * -20~20: "陌生同门"
      * 21~40: "关注的师姐"  
      * 41~60: "暗生情愫"
      * 61~80: "确认心意"
      * 81~100: "生死相依"
```

## 总结

转阶段人设的核心：
1. **好感度分阶段**：清晰的数值区间
2. **态度渐变化**：每阶段态度明确不同
3. **对话差异化**：语气、称呼、句式都要变
4. **触发条件化**：用世界书条件或脚本自动切换
5. **保持一致性**：核心性格贯穿始终

让红颜角色有血有肉，情感线自然流畅！✨
